Reports a concrete class/struct/annotation declaration that does not implement every abstract method it inherits (transitively — inheritance is always virtual in K) from its base interfaces or abstract classes.

The class name is highlighted. Two quick fixes are offered: generate override stubs for every missing method, or mark the class itself abstract.

Skipped (fail-soft): interface declarations (implicitly abstract), aggregates already marked abstract, and template/generic aggregate definitions (parametrised member set). An interface member carrying the default specifier already has a body and does not require an override.
