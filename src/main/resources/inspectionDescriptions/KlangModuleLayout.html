Reports directory-level module-layout errors of the default per-directory module policy.

Within a directory, if at least one .k file declares no module, the other files must all declare the same module — otherwise the directory is ambiguous (mixed directory). If every file declares a module, the directory may host several modules.

A directory that declares no module inherits the module of the nearest ancestor directory that declares exactly one. If that ancestor declares several modules, the inherited module is ambiguous.

Resolution still degrades gracefully (a deterministic best-effort module is used); this inspection only surfaces the layout problem so it can be fixed by declaring a single, explicit module.
