Reports references that cannot be resolved to any declaration — the K spec's Undefined symbol error (§16).

Covers names used as expressions, qualified names in type / directive positions (typeSpec, base, using, friend, @annotation, …) and member accesses (a.b, a->b).

The check is fail-soft to avoid false positives:

  • fundamental types and keywords carry no reference and are never flagged;
  • deferred resolution (§16): a name in callee position (Name(args) or recv.member(args)) is left untouched because it may legitimately be a unified-call-syntax or overload candidate;
  • nothing is reported while project indexes are still building (dumb mode).

When an unresolved simple name is declared by another module, an Add import quick fix proposes the module(s) to import.

Note: cross-module / external-symbol resolution (EXTERNAL_LOOKUP, §5.8) is not yet fully modelled, so a symbol living in an un-imported external module is reported as unresolved and relies on the add-import quick fix.
